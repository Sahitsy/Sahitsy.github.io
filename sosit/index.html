Projet [SOS-IT]

Spécifications fonctionnelles

SOS-IT est un système de gestion de tickets qui a pour objectif de gérer le traitement et le suivi des différentes anomalies, d'une application informatique, détectées avant ou après sa mise en production. On peut aussi utiliser SOS-IT pour demander le rajout d'une nouvelle fonctionnalité à une application existante.

L'application permettra à l'utilisateur de : créer un ticket, rechercher un ticket existant, prendre en charge un ticket, l'affecter à une équipe ou à une personne. La personne en charge du ticket informe le demandeur de l'état de son avancement et peut clôturer le ticket s'il est résolu (le demandeur pourra aussi clôturer le ticket qu'il a crée si besoin). Les différents intervenants peuvent recevoir des notifications sur l'état d'avancement du ticket. Maquette du projet.

Spécifications téchniques

Langage de programmation : C#
Application Web : Razor Pages
Accès aux données : Entity Framework
Base de données : SQL Server
Hébergement : Microsoft Azure

Un ticket c'est : N° du ticket, Date de création, Titre, Type (demande de correction, rajout d'une fonctionnalité), Description (+ Pièce jointe, Capture d'écran, ...), Assigné à ..., Priorité (Haute, Moyenne, Faible), Etat (Ouvert, Pris en charge, Résolu).

Rôles des utilisateurs : Administrateur, Développeur, Testeur, Client.

Administration : Ajouter un système d'identification Login + Mot de passe. Pour les tests ajouter un DemoUser, qui peut choisir son propre rôle.
